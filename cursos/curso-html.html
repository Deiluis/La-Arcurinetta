Curso de HTML 5

Algo de historia sobre HTML

El origen

En 1980, Tim Berners-Lee, un fisico e ingeniero britanico empleado del CERN (Centro Europeo de Fisica Nuclear) propuso un nuevo sistema de hipertexto para compartir documentos. Tras finalizarlo, lo presento en un concurso en la empresa para solucionar el problema de la comunicacion; presentó el mismo junto con el lenguaje para escribirlo: HTML o HyperText Markup Language y el primer documento formal escrito en este lenguaje, cuyo nombre inicialmente fue HTML tags, o etiquetas HTML. Se ha hecho una recreación y se puede visualizar desde navegadores modernos haciendo click acá.

¿Cómo siguio?

En 1996 se creo el W3C o World Wide Web Consortium como organismo que se encargaría de establecer los estandares generales para HTML y su correcto funcionamiento en la red. La W3C publica posteriormente HTML 3.2 en 1997, que posee applets de Java para el texto y las imagenes. Luego HTML 4.0 en 1998, y HTML 4.01, una mejora de este ultimo, en 1999.

Desde la publicación de HTML 4.01, el W3C se centro principalmente en el desarrollo del estandar XHTML, hasta el 2008 cuando retomaron el desarrollo del estandar HTML, para finalmente publicar HTML 5 en 2014.

Ahora que ya sabemos un poco más sobre la historia de este lenguaje, veamos sus contenidos.

Contenidos

Elementos HTML

La base de todo en HTML son los elementos. Un elemento es una especie de "caja" que el navegador dibuja en nuestra pantalla; estas cajas componen toda la estructura HTML.

Ya sabiendo que es un elemento, debemos saber que este tiene ciertas partes. Las partes que conforman al elemento son:

  • La etiqueta de apertura.
  • El contenido.
  • La etiqueta de cierre.

Estas partes son fragmentos del código que nos permiten crear elementos. El primer elemento que veremos como ejemplo es <h1> que nos permite escribir el encabezado principal de la página.

Al crear un elemento h1, debemos escribirlo de la siguiente manera: <h1>Hola mundo</h1>. En este caso, <h1> es la etiqueta de apertura, la cual se escribe con los signos de menor y mayor (< y >) y el nombre de la etiqueta en medio, quedando <h1>. Luego de la etiqueta de apertura se introduce el contenido deseado que se muestre en pantalla, que en este caso es Hola mundo, pero puede ser cualquier texto, o incluso otros elementos, pero veremos eso más adelante. Para finalizar se escribe la etiqueta de cierre, la cual es similar a la etiqueta de apertura, pero esta contiene una barra inclinada o slash antes del nombre de la etiqueta, quedando: </h1>.

Estructura básica de HTML

Para que nuestro navegador pueda interpretar nuestro código HTML debemos cumplir dos condiciones primero:

  1. Nuestro archivo debe ser formato HTML (que sea terminado en .html).
  2. Nuestro archivo debe contener los elementos <!DOCTYPE html>, <html>, <head> y <body>.

El elemento <!DOCTYPE html> le indica al navegador que versión de HTML se esta utilizando. El elemento <html> el elemento de mayor jerarquía en nuestra página, ya que contiene a los elementos <head> y <body>. El elemento <head> o "cabeza" contiene toda la información externa pero relevante de la página, tal como el titulo, el icono, los elementos <meta> para la codificación, el lenguaje, entre otras cosas que le ayudan al navegador a posicionar nuestra página web en los motores de búsqueda. Por último, el elemento <body> o "cuerpo" contiene todos los elementos visibles de nuestra página, los encabezados, párrafos, enlaces, entre otros que iremos viendo a lo largo del curso.

Encabezados

Como mencionamos anteriormente, el elemento <h1> sirve para introducir el encabezado o título principal de la página, pero como tal, existen cinco elementos más que cumplen esta función, estos son: <h2>, <h3>, <h4>, <h5>, y <h6>. La principal diferencia entre cada uno de estos elementos es su su función como título.

Por ejemplo: <h1> se utiliza para el título principal de la página, y solo debe introducirse uno por documento, ya que cómo se dijo, es el título PRINCIPAL, y escribir más de uno podría confundir al navegador y posicionar mal nuestra página en la web. Siguiendo con los elementos, <h2> se utiliza como subtitulos, o titulos de secciones, <h3> como subtitulos de secciones, por último <h4>, <h5> y <h6> se utilizan para subtitular componentes de menor importancia.

Cabe remarcar el cuidado que se debe tener en la repetición de los elementos de titulo según su jerarquía (mientras más alta sea esta, menos veces debe utilizarse), como también se debe tener en cuenta cuando usar cada uno basado en la función descrita anteriormente, y no basarse en el tamaño de letra, ya que de guiarnos de esta manera, podemos posicionar de mala manera nuestro sitio web, tal como se explico anteriormente.

Párrafos y modificadores de texto

Ya hemos visto elementos de encabezado para los títulos, ahora es el turno de los elementos de párrafo y los modificadores del texto.

Para iniciar un nuevo párrafo en nuestra página, se utiliza el elemento <p>. Tal como los elementos de encabezado, este requiere de una etiqueta de apertura y de cierre; entre estas dos ira el contenido que queremos mostrar en nuestro párrafo. Cabe aclarar que se debe utilizar un elemento de párrafo por cada párrafo, y no introducir todo nuestro texto en un mismo párrafo aplicando saltos de línea.

También existen los elementos modificadores de texto, los cuales modifican visualmente el texto que este encerrado entre ellos. Algunos de ellos son:

  • El elemento <b>, de bold o negrita.
  • El elemento <i>, de italic o italica.
  • El elemento <strong>, en negrita con una importancia semántica.
  • El elemento <em>, en italca con una importancia semántica.
  • El elemento <small>, o pequeño.
  • El elemento <strike>, o tachado.
  • El elemento <span>, o separador de texto invisible.

Para utilizar cualquiera de estos modificadores, unicamente se debe encerrar entre sus etiquetas el texto que se quiere modificar. Habrán notado que los elementos <strong> y <em> son exactamente iguales a los elementos <b> e <i>. La principal diferencia es que <strong> y <em> tienen importancia "semántica", pero más adelante en el curso se tratará ese topico.

Enlaces

Ya sabemos insertar títulos, párrafos, e incluso modificar textos, pero ahora surge una duda: ¿Hay alguna manera simple de navegar entre páginas de una misma web? Para solucionar esto existen los elementos de enlace, representados con los elementos <a>.

Los elementos <a> funcionan como enlaces a otros documentos HTML, y para su correcto funcionamiento se escriben de la siguiente manera: <a href=#>Click acá</a>. Cómo podemos observar, estos elementos hacen uso de un componente nuevo para nosotros, este componente son los atributos de los elementos y los elementos sensibles.

Los atributos son una serie de parametros que definen o modifican al elemento, y cada debe tener un valor. Estos atributos deben definirse en la etiqueta de apertura del elemento, debe escribirse el nombre del atributo, un signo igual (=) y el valor del atributo entre comillas (""). En este caso, el elemento <a> utiliza el atributo href, el cual tiene un valor de #, pero este valor puede ser cambiado; distintos valores en ciertos atributos pueden modificar el funcionamiento de los elementos a los cuales son aplicados. Por ejemplo:

  • Si se utiliza un númeral (#) y un determinado id correspondiente a un elemento del mismo documento, se puede acceder hasta dicho elemento.
  • Si se utiliza una ruta de acceso a un archivo local, se puede acceder hasta el mismo.
  • Si se utiliza una url de otra página web, se puede acceder a la misma. Si ademas se añade el atributo target con el valor "_blank" se puede acceder a esta nueva página, abriendo una nueva pestaña para la misma.
  • Si solo se utiliza un númeral (#) como en el primer ejemplo, no se realiza ninguna acción.
  • Si no se coloca ningun valor, el elemento <a> no posee un "elemento sensible".

El elemento sensible esta conformado por el contenido del elemento <a>. Es este mismo el que ejecuta la operación de redirección al hacer click o tocar en él, puede ser texto plano, o incluso otros elementos.

Atributos class y id

En el capitulo anterior se explicó el concepto de un atributo y se tomo de ejemplo el atributo href. Sin embargo, ese atributo es especifico de los elementos <a>. Existen otros atributos denominados "globales", los cuales pueden ser aplicados a cualquier elemento y funcionan de una manera general. Dos de estos son los atributos class e id.

Tanto class como id son atributos que sirven para identificar elementos. Sin embargo, la principal diferencia entre estos es la cantidad de elementos que pueden ser referenciados con estos atributos. Por ejemplo: class puede utilizar un mismo valor para identificar a un cierto grupo de elementos de una misma clase, mientras que id solo puede utilizar un mismo valor una sola vez, para identificar un solo elemento especifico.

Algo adicional a mencionar: Si a un elemento se le añade un id, este puede ser ubicado utilizando enlaces, tal como se mostró en los ejemplos del capítulo anterior.

Imágenes

Hasta ahora hemos trabajado puramente con texto, enlaces, y modificadores de los mismos, en este capitulo veremos como utilizar imágenes. Para insertar una imagen se utiliza el elemento <img>, cuya sintaxis para su correcto funcionamiento es: <img src="imagen.png">.

Viendo el elemento <img> podremos notar dos cosas principales:

  1. No posee etiqueta de cierre.
  2. Utiliza un nuevo atributo: src.

Los elementos <img> como varios otros no poseen etiqueta de cierre, por lo que se les considera elementos "abiertos." En adición a lo anterior, podemos ver que el nuevo atributo src (de source, o fuente en ingles) debe contener la ruta exacta de la imagen a utilizar. Tal como en los enlaces, la ruta puede ser local (nuestra misma PC) o remota (desde internet) pero siempre debe contener la extensión de la imagen para poder funcionar correctamente.

Elementos en línea y en bloque

En este capitulo se tratará un más tema conceptual: Los elementos en línea y los elementos en bloque.

Existen dos maneras principales de mostrar los elementos en el navegador: en línea, o en un bloque. Los elementos en linea, al dibujarse, unicamente toman el espacio que ocupan fisicamente: si ocupa 30 píxeles de ancho, solo toma 30 píxeles del ancho total de la pantalla. Por otro lado, los elementos en bloque toman el espacio de toda la linea que ocupan. Por ejemplo, si el elemento ocupa 30 píxeles de ancho, aún así tomara todo el espacio de la línea, obligando al navegador a dibujar el resto de elementos en las lineas siguientes.

Algunos elementos en línea son los <a>, <b>, <i>, <strong>, <em>, <img>, entre otros. Algunos elementos en bloque son los <h1>-<h6>, <p>, <div>, <br>, entre otros.

Elementos de lista

En este capítulo se aboradará un nuevo tema: Las listas de elementos.

Para crear una lista de elementos, existen dos tipos de elementos de lista distintos: <ol> (de ordered list, o lista ordenada) y <ul> (de unordered list, o lista desordenada). La principal diferencia entre ambas son el tipo de items que utilizan, <ol> utiliza un formato numerado, desde el 1 hasta el número máximo que se utilice. Por el otro lado, <ul> utiliza un formato de puntos negros o viñetas como items, sin números ni un orden especifico.

En ambos casos, para armar la lista, debemos introducir dentro de la etiqueta <ol> o <ul> un nuevo elemento, el item de lista o elemento <li> (list item). Dentro de los items es donde se incluye el contenido a enlistar, ya sean textos u otros elementos.

HTML semántico

En capítulos anteriores se mencionó el tema del HTML semántico. En este capítulo se ahondará más sobre el tema; conoceremos el concepto, y los elementos semánticos.

Para el posicionamiento de las páginas web en los navegadores, los motores web utilizan una herramienta llamada SEO o Search Engine Optimization. Esta se encarga de revisar y comprobar que los documentos HTML mantengan un cierta lógica y una buena semántica; tal cómo un texto debe poseer coherencia y cohesión, una página web debe mantener una buena semántica.

Para mantener una buena semántica, se utilizan los elementos semánticos. Estos son elementos que tienen un rol especifico asignado al documento. Por ejemplo, los elementos <h1> a <h6>, que tienen un rol o función especifica unicamente como títulos. Existen varios elementos semánticos, estos son:

  • El <header> o encabezado.
  • El <nav> o barra de navegación.
  • El <main> o contenido principal.
  • El <section> o sección de contenido.
  • El <article> o artículo de la sección.
  • El <aside> o contenido adicional o publicitario que se ubica a un costado.
  • El <footer> o pie de página.
  • Los <h1> a <h6> o los titulos.
  • El <strong> o resaltado en negrita.
  • El <em> o resaltado en italica.

El buen uso y posicionamiento de los elementos semánticos nos garantizarán una buena semántica en nuestro sitio web.
